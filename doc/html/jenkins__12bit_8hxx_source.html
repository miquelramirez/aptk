00013 --------------------------------------------------------------------
00014 mix -- mix 3 32-bit values reversibly.
00015 For every delta with one or two bits set, and the deltas of all three
00016   high bits or all three low bits, whether the original value of a,b,c
00017   is almost all zero or is uniformly distributed,
00018 * If mix() is run forward or backward, at least 32 bits in a,b,c
00019   have at least 1/4 probability of changing.
00020 * If mix() is run forward, every bit of c will change between 1/3 and
00021   2/3 of the time.  (Well, 22/100 and 78/100 for some 2-bit deltas.)
00022 mix() was built out of 36 single-cycle latency instructions in a 
00023   structure that could supported 2x parallelism, like so:
00024       a -= b; 
00025       a -= c; x = (c>>13);
00026       b -= c; a ^= x;
00027       b -= a; x = (a<<8);
00028       c -= a; b ^= x;
00029       c -= b; x = (b>>13);
00030       ...
00031   Unfortunately, superscalar Pentiums and Sparcs can't take advantage 
00032   of that parallelism.  They've also turned some of those single-cycle
00033   latency instructions into multi-cycle latency instructions.  Still,
00034   this is the fastest good hash I could find.  There were about 2^^68
00035   to choose from.  I only looked at a billion or so.
00036 --------------------------------------------------------------------
00037 */
00038 #define jenkins_mix(a,b,c) \
00039 { \
00040   a -= b; a -= c; a ^= (c>>13); \
00041   b -= c; b -= a; b ^= (a<<8); \
00042   c -= a; c -= b; c ^= (b>>13); \
00043   a -= b; a -= c; a ^= (c>>12);  \
00044   b -= c; b -= a; b ^= (a<<16); \
00045   c -= a; c -= b; c ^= (b>>5); \
00046   a -= b; a -= c; a ^= (c>>3);  \
00047   b -= c; b -= a; b ^= (a<<10); \
00048   c -= a; c -= b; c ^= (b>>15); \
00049 }

00053 hash() -- hash a variable-length key into a 32-bit value
00054   k       : the key (the unaligned variable-length array of bytes)
00055   len     : the length of the key, counting by bytes
00056   initval : can be any 4-byte value
00057 Returns a 32-bit value.  Every bit of the key affects every bit of
00058 the return value.  Every 1-bit and 2-bit delta achieves avalanche.
00059 About 6*len+35 instructions.
00060 
00061 The best hash table sizes are powers of 2.  There is no need to do
00062 mod a prime (mod is sooo slow!).  If you need less than 32 bits,
00063 use a bitmask.  For example, if you need only 10 bits, do
00064   h = (h & hashmask(10));
00065 In which case, the hash table should have hashsize(10) elements.
00066 
00067 If you are hashing n strings (ub1 **)k, do it like this:
00068   for (i=0, h=0; i<n; ++i) h = hash( k[i], len[i], h);
00069 
00070 By Bob Jenkins, 1996.  bob_jenkins@burtleburtle.net.  You may use this
00071 code any way you wish, private, educational, or commercial.  It's free.
00072 
00073 See http://burtleburtle.net/bob/hash/evahash.html
00074 Use for hash table lookup, or anything where one collision in 2^^32 is
00075 acceptable.  Do NOT use for cryptographic purposes.
